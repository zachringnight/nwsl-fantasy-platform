Reference the master context document for locked decisions.
Reference Step 1 for target users and use cases.

---

## Your task

Map the ideal user flows for every critical action. Optimize for minimum confusion and minimum taps on mobile.

---

## Deliver exactly these flows

For each flow, provide:
- **Entry point** (where the user starts)
- **Steps** (numbered, with the screen or component at each step)
- **Decision points** (where the user chooses between paths)
- **Tap count on mobile** (from entry to completion)
- **Friction risks** (where users might get confused or drop off)
- **Design mitigation** (how you prevent the friction)

### Required flows

1. **First visit to sign up** (landing page through account creation)
2. **Create a league** (from dashboard through shareable invite link)
3. **Join a league** (via invite link and via league code)
4. **Enter draft room** (from league home through draft lobby to live draft)
5. **Draft players** (picking, queuing, autopick toggle, clock management)
6. **Set lineup** (from league home through lineup confirmation)
7. **Swap a player before lock** (quick-swap from lineup view)
8. **Track live scoring during a match** (matchup view with real-time updates)
9. **Review standings** (league standings and playoff picture if applicable)
10. **Add/drop a player** (find player, drop bench player, confirm)
11. **Submit a waiver claim** (priority selection, conditional drop)
12. **Commissioner edits** (change a league setting mid-season)
